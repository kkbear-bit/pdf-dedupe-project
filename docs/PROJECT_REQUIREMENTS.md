# PDF Dedupe 项目需求说明书

## 1. 项目定位

PDF Dedupe 是一款面向科研人员、研究助理、研究生、知识管理用户和文献库维护者的本地化 PDF 去重工具。项目目标不是简单删除重复文件，而是构建一套低风险、可审计、可交付的文献清理流程，让用户能够在不依赖云端服务、不上传敏感资料的前提下，完成论文 PDF 文件夹的快速整理。

该工具优先服务以下典型场景：

- Zotero、EndNote、Mendeley 等文献管理工具导出的 PDF 文件夹清理。
- 课题组共享资料、综述写作资料、系统评价初筛文献的本地去重。
- 批量下载文献后产生的重复 PDF、`.compare.pdf` 文件和临时副本清理。
- 投稿、归档、交接前生成一份干净、可复查的 PDF 结果目录。

## 2. 产品愿景

项目希望成为“科研文献整理的最后一公里工具”：轻量、透明、可控，不要求用户安装大型软件，也不改变原始文献库结构。工具应当像一个可靠的文献整理助手，先把风险暴露给用户，再由用户确认执行，最终留下清晰的结果目录、隔离目录和操作报告。

长期目标是从单一脚本演进为一个可扩展的文献资产治理工具，逐步支持递归扫描、相似文件识别、元数据报告、GUI 界面和跨平台打包。

## 3. 目标用户

| 用户类型 | 核心诉求 | 使用特征 |
| --- | --- | --- |
| 科研人员 | 快速清理重复论文 PDF | 不希望学习复杂命令 |
| 研究生 | 组会、开题、综述前整理文献 | 文件夹来源多、命名混乱 |
| 文献管理员 | 生成可交付的清理结果 | 重视报告和可追溯性 |
| 系统评价团队 | 初筛前整理下载结果 | 文件量大，需要低误删风险 |

## 4. 核心问题

当前文献 PDF 去重常见痛点包括：

- 文件名不统一，单靠文件名无法判断是否重复。
- 直接删除重复文件风险高，误删后难以恢复。
- 去重过程缺乏报告，无法解释“删了哪些、保留了哪些”。
- 部分下载器会生成 `.compare.pdf` 等临时或对比文件，干扰归档。
- 许多用户只需要一个能双击运行的轻量工具，而不是完整文献管理系统。

## 5. 核心目标

### 5.1 功能目标

- 支持对指定文件夹内 PDF 文件进行非递归扫描。
- 使用 SHA256 哈希识别完全一致的 PDF 文件。
- 默认识别 `.compare.pdf` 后缀文件作为候选重复项。
- 默认以预览模式运行，明确展示将要处理的文件。
- 执行模式下将重复候选移动到隔离目录，而不是直接删除。
- 生成 CSV 报告，记录原路径、隔离路径、处理原因、保留文件和哈希值。
- 生成日期命名的干净结果目录，便于交付和复查。
- 提供命令行入口和双击启动入口。

### 5.2 非功能目标

- 本地运行，不上传 PDF，不依赖第三方网络服务。
- 零第三方依赖，仅要求 Windows 和 PowerShell。
- 操作过程可撤销，隔离目录保留被移动文件。
- 输出结构稳定，便于后续自动化或人工审阅。
- 对非技术用户友好，对高级用户保留命令行可控性。

## 6. 当前版本范围

当前版本聚焦“安全、轻量、可用”的最小闭环：

- 仅扫描目标文件夹第一层 PDF 文件。
- 仅识别内容完全一致的重复文件。
- 仅对 `.compare.pdf` 做规则化候选识别。
- 不解析 PDF 标题、DOI、作者、页码等语义元数据。
- 不自动删除任何文件。

## 7. 用户流程

### 7.1 双击启动流程

1. 用户双击 `启动PDF去重.bat` 或 `start_pdf_dedupe.bat`。
2. 程序提示输入待处理 PDF 文件夹。
3. 程序提示输入输出父目录，可留空使用默认目录。
4. 程序执行预览，列出重复候选和处理原因。
5. 用户确认输入 `Y`。
6. 程序移动重复候选到隔离目录。
7. 程序生成 CSV 报告。
8. 程序复制保留 PDF 到干净结果目录。

### 7.2 命令行流程

```powershell
.\dedupe-pdfs.ps1 -TargetDir "D:\papers"
.\dedupe-pdfs.ps1 -TargetDir "D:\papers" -OutputDir "D:\output" -Apply
```

## 8. 输出规范

执行后应产生以下结构：

```text
目标文件夹/
  _pdf_duplicates_quarantine/
    20260610_153000/
      duplicate-a.pdf
      duplicate-b.compare.pdf
    dedupe_report_20260610_153000.csv

输出父目录/
  PDF_deduped_result_20260610/
    kept-a.pdf
    kept-b.pdf
```

CSV 报告字段：

| 字段 | 说明 |
| --- | --- |
| `OriginalPath` | 被移动文件的原始路径 |
| `QuarantinePath` | 隔离后的文件路径 |
| `Reason` | 处理原因 |
| `Keeper` | 对应保留文件路径 |
| `Hash` | SHA256 哈希值 |

## 9. 风险控制

- 默认预览，不主动修改文件。
- 执行前需要用户显式确认。
- 重复文件移动到隔离目录，不做永久删除。
- 隔离目录按时间戳分批，避免覆盖历史操作。
- 输出目录使用日期命名，降低误覆盖风险。
- 对同名隔离文件自动追加短 GUID，避免文件名冲突。

## 10. 成功标准

项目达到以下标准即可视为当前阶段完成：

- 用户可以在无额外依赖的 Windows 环境中运行。
- 对完全一致的 PDF 重复文件能够稳定识别。
- 对 `.compare.pdf` 文件能够按规则纳入候选。
- 预览、确认、隔离、报告、结果复制形成闭环。
- GitHub 仓库包含清晰 README、许可证、忽略规则和需求文档。

## 12. 设计原则

- 安全优先：宁可多提示，也不隐式删除。
- 可解释优先：每个处理动作都应有原因。
- 本地优先：文献资产不离开用户电脑。
- 轻量优先：避免引入不必要的运行时和依赖。
- 渐进增强：先做好精确去重，再扩展智能能力。
